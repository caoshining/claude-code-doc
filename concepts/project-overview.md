# 项目概述

## 📌 项目简介

**Claude Code Best (CCB)** 是一个反向工程复刻 Anthropic 官方 Claude Code CLI 的开源项目。

## 🎯 项目目标

1. **复刻官方功能** - 完整恢复 Claude Code 核心能力
2. **开源透明** - 让内部实现可审计、可学习
3. **扩展能力** - 添加官方企业版才有的特性
4. **社区驱动** - 开源协作改进

## 🔍 项目背景

### 官方 Claude Code

Anthropic 官方推出的 Claude Code CLI 是一个强大的终端 AI 编程助手，但：
- 闭源，内部实现不透明
- 部分功能仅企业版可用
- 无法深度定制

### CCB 的定位

- ✅ **完全开源** - 所有代码公开
- ✅ **功能对等** - 实现官方所有核心功能
- ✅ **扩展增强** - 添加官方没有的特性
- ✅ **可自托管** - 支持完全离线部署

## 🚀 核心特性

### 1. 基础能力 (与官方对等)

- 📝 **智能对话** - Claude API 集成
- 🔧 **工具调用** - 60+ 内置工具
- 📁 **项目理解** - 代码库上下文感知
- 🎨 **终端 UI** - React Ink 渲染
- 💾 **会话管理** - 持久化对话历史

### 2. 扩展特性 (超越官方)

| 特性 | 说明 | 官方支持 |
|------|------|----------|
| **Goal 持续驱动** | `/goal` 跨轮自动完成任务 | ❌ 企业版 |
| **Artifacts** | HTML 上传托管 | ❌ |
| **Ultracode** | 多 Agent 确定性编排 | ❌ |
| **群控技术** | Pipe IPC + LAN 发现 | ❌ |
| **ACP 协议** | IDE 深度集成 | ❌ |
| **穷鬼模式** | 减少并发节省成本 | ❌ |
| **自托管** | 完全离线部署 | ❌ |

## 📦 技术选型

### 为什么选 Bun?

- ⚡ **性能** - 比 Node.js 快 3-4 倍
- 📦 **内置** - 原生支持 TypeScript、测试
- 🔧 **工具链** - 统一构建、运行、打包

### 为什么选 React Ink?

- 🎨 **组件化** - React 生态复用
- 🖥️ **终端渲染** - 跨平台 TUI
- 🔌 **插件友好** - 易于扩展
