# 核心循环 (Core Loop)

## 🔄 什么是核心循环?

核心循环是 CCB 的心脏，负责处理每次用户输入并生成响应。它是用户与 AI 交互的主循环。

## 🔑 核心组件详解

### 1. REPL.tsx - 用户界面

**职责**: React Ink 组件，渲染终端 UI

**关键功能**:
- 渲染消息历史
- 处理用户输入
- 显示工具权限提示
- 键盘快捷键处理
- 成本/Token 显示

**状态管理**:
```typescript
interface REPLState {
  messages: Message[]
  input: string
  isProcessing: boolean
  tools: Tools
  // ...
}
```

### 2. QueryEngine.ts - 编排器

**职责**: 高级编排逻辑，管理回合级状态

**关键功能**:
- **会话管理**: sessionId、turn 计数
- **压缩决策**: 检查是否需要压缩上下文
- **文件历史**: 快照文件状态用于归因
- **Token 预算**: 管理 output token 预算
- **消息归因**: 追踪哪次修改引入了变化

**核心方法**:
```typescript
class QueryEngine {
  async query(userInput: string): Promise<QueryResult>
  private async buildRequest(): Promise<RequestParams>
  private async processResponse(): Promise<void>
  private shouldCompact(): boolean
  private updateFileHistory(): void
}
```

### 3. query.ts - API 执行

**职责**: 直接与 Claude API 交互

**关键功能**:
- 构建请求参数
- 调用 Messages API
- 处理流式响应
- 执行工具调用
- 错误重试

**工具调用流程**:
```
检测 tool_use_block
    ↓
查找对应 Tool
    ↓
请求用户权限 (如需要)
    ↓
执行 Tool
    ↓
收集 tool_result
    ↓
继续 API 调用 (带结果)
```

## 🔄 Turn Loop (回合循环)

一个完整的回合包括:

1. **输入阶段**: 用户提交问题/命令
2. **请求阶段**: 构建并发送 API 请求
3. **响应阶段**: 接收并处理流式响应
4. **工具阶段**: 执行工具调用
5. **更新阶段**: 更新内部状态
6. **渲染阶段**: 显示结果给用户

### 多轮工具调用

当 AI 需要调用多个工具时:

```
User → API → Tool 1 → Result 1 → API → Tool 2 → Result 2 → ... → Final Response
```

每个工具调用都是一个新的 API 请求，携带之前的结果。

## 🧊 压缩检查点

QueryEngine 在每个回合后检查是否需要压缩:

```typescript
// 检查条件
- 总 token 数接近窗口上限
- 最近响应超过 200k token
- 启用自动压缩且达到阈值
```

如需压缩，调用 `compactMessages()` 保留:
- 最近 N 条消息
- 系统提示
- 关键上下文

## 📊 Token 预算

QueryEngine 管理 token 使用:

```typescript
interface TokenBudget {
  totalInput: number      // 总输入 token
  totalOutput: number     // 总输出 token
  remaining: number       // 剩余预算
  warningThreshold: number // 警告阈值
}
```

**预算策略**:
- 默认: Claude 模型的 context 窗口
- 可通过环境变量覆盖
- 超限时触发压缩或拒绝请求
