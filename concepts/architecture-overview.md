# 架构总览

## 🏗️ 整体架构

CCB 采用**分层架构**设计，从入口到 UI 共 8 个层级：

```
┌─────────────────────────────────────────────────────────┐
│                    用户交互层                              │
│                  Terminal / Browser                       │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                     入口层 (Entry)                        │
│              cli.tsx → main.tsx (Commander.js)           │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   核心循环层 (Core Loop)                   │
│         QueryEngine → query → REPL (Turn Loop)            │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                     UI 层 (React Ink)                     │
│              149 Components + @anthropic/ink             │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   工具层 (Tool System)                    │
│           tools.ts → Tool.ts → 60+ Tools                  │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   服务层 (Services)                       │
│   api/ | mcp/ | compact/ | acp/ | analytics/ | goal/     │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                  状态层 (State Management)                │
│             AppState + Zustand + bootstrap/state           │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                基础设施层 (Infrastructure)                 │
│      17 Workspace Packages | Vendor Binaries              │
└─────────────────────────────────────────────────────────┘
```

## 📦 Monorepo 结构

项目使用 **Bun workspaces** 管理多个包：

### 核心包 (Core)

```
@ant/
├── ink/                    # Forked Ink 框架
├── computer-use-mcp/       # Computer Use MCP 服务
├── computer-use-input/     # 键鼠模拟
├── computer-use-swift/     # 截图 + 应用管理
├── claude-for-chrome-mcp/  # Chrome 控制
└── model-provider/         # Model provider 抽象

```

## 🔄 数据流

### 请求流程

```
用户输入
   ↓
REPL.tsx (渲染输入框)
   ↓
QueryEngine.ts (编排请求)
   ↓
query.ts (构建 API 调用)
   ↓
API Provider (发送请求)
   ↓
Claude API (流式响应)
   ↓
query.ts (处理流、工具调用)
   ↓
Tool.ts (执行工具)
   ↓
QueryEngine.ts (更新状态)
   ↓
REPL.tsx (渲染结果)
```

### Turn Loop (回合循环)

```
      ┌─────────────────┐
      │  用户输入/响应   │
      └────────┬────────┘
               ↓
      ┌─────────────────┐
      │  构建请求消息   │
      └────────┬────────┘
               ↓
      ┌─────────────────┐
      │  API 调用       │
      └────────┬────────┘
               ↓
      ┌─────────────────┐
      │  流式处理       │
      │  - 工具调用     │
      │  - 内容生成     │
      └────────┬────────┘
               ↓
      ┌─────────────────┐
      │  状态更新       │
      └────────┬────────┘
               ↓
      ┌─────────────────┐
      │  UI 渲染         │
      └────────┬────────┘
               │
               └──→ 回合结束
                    ↓
              检查是否继续
                    ↓
              ┌─────┴─────┐
              │           │
           是            否
              │           │
              ↓           ↓
         继续下一回合   等待用户
```

## 🔑 关键设计决策

### 1. 为什么分层?

- **关注点分离** - 每层专注特定职责
- **可测试性** - 层间接口明确
- **可扩展性** - 新功能易于插入

### 2. 为什么 Monorepo?

- **代码共享** - 工具、类型复用
- **统一构建** - 依赖管理简单
- **原子提交** - 跨包修改一次提交

### 3. 为什么代码分割?

- **内存优化** - 按需加载降低 RSS
- **启动速度** - 减少解析时间
- **缓存友好** - 小文件缓存命中率高

## 📊 模块依赖关系

```
cli.tsx (入口)
  └── main.tsx (CLI 定义)
       └── REPL.tsx (UI)
            ├── QueryEngine.ts (编排)
            │    └── query.ts (API)
            │         ├── tools.ts (工具)
            │         ├── services/api/ (API)
            │         └── services/compact/ (压缩)
            └── components/ (UI 组件)
                 └── @anthropic/ink (框架)
```
